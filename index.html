メタ言語を使おう！

最近では HTML とか CSS とか JavaScript をクソマジメに書いていると、バカにされる風潮がある。

そういえば、SCSS とかいう変な CSS が巷では流行っているらしい。

なんでも、「メタ言語」というモノを使って書くのがイマドキの新常識なんだとか。

メタ言語とは

CoffeeScript

こう書くと

name = 'メタ言語'
$('a').on 'click', (ev) ->
  alert("こんにちは、\#{name}。")

こうなる

var name = 'メタ言語';
$('a').on('click', function (ev) {
  alaert('こんにちは、' + name + '。');
});

Stylus

こう書くと

black = rgb(0,0,0);
white = rgb(255,255,255);
body
  background-color: black
  color: white

こうなる

body {
  background-color: #000;
  color: fff;
}

Jade

こう書くと

!!! 5
html
  body
    h1 おれのホームページ

こうなる

<!DOCTYPE html>
<html>
  <body>
    <h1>おれのホームページ</h1>
  </body>
</html>

元の言語よりシンプルに書けるのが「メタ言語」というものらしい。

メタ言語の特徴

  • 元の言語より短く書ける
  • 元の言語にはない便利な機能が追加されている。
  • 構造を括弧 { ... } ではなく、インデントで表現するモノが多い。
  • あくまで元の言語に変換するための「メタ」言語という立ち位置。
  • 割と流行っているらしい。

「元の言語に変換する」...?

メタ言語の使い方

インストール

npm install coffee-script stylus jade -g

変換コマンド

CoffeeScript

coffee /path/to/script.coffee

Stylus

stylus /path/to/style.styl

Jade

jade /path/to/page.jade

毎回これらのコマンドを打つのは面倒なので、Makefile の仕組みに習った Cakefile をつくると良いらしい。

結局手間を増やして何が嬉しいのか。

WHY

たとえば CoffeeScript だったら

  • 初心者が js そのままを書くより安全で早いコードが書ける。
  • バグを入れこみ易い部分を回避する仕組みがある。
  • オブジェクト指向をより直感的に利用できる。
  • インデントによる構造化で書き手の自己主張を最小限に留め、誰が見ても同じ書き方、読みやすいコードを共有できる。

たとえば Stylus だったら

  • 変数・四則演算・ミックスイン等、プログラミング的な DRY の要素を使って、メンテナンスし易い CSS が書ける。
  • 出力される CSS はミニファイ化したものを出力する事も可能なので、プロダクトのパフォーマンスチューニングにも繋がる。
  • インデントによる構造化で書き手の自己主張を最小限に留め、誰が見ても同じ書き方、読みやすいコードを共有できる。

たとえば Jade だったら

  • サーバーサイドテンプレートエンジンとしても使える。
  • 閉じタグがないので閉じ忘れの心配がない。
  • インデントによる構造化で書き手の自己主張を最小限に留め、誰が見ても同じ書き方、読みやすいコードを共有できる。

手間をかけるメリットは十分にあるし、手間を最小限にする仕組みもある。
